## 리다이렉트 시 Model 값 유실 문제

### 문제
장바구니 추가 과정에서 수량 검증에 실패한 뒤 상품 상세 페이지로 리다이렉트할 때, 검증 결과가 함께 전달되지 않는 문제가 있었습니다.  
그 결과 에러 메시지나 검증 상태가 화면에 정상적으로 표시되지 않았습니다.

### 해결
원인을 확인한 결과, `Model`은 요청 범위에서만 유효하기 때문에 리다이렉트 상황에서는 값이 유지되지 않았습니다.  
이를 `RedirectAttributes`의 `addFlashAttribute()`로 변경해, 리다이렉트 이후에도 필요한 검증 결과가 유지되도록 수정했습니다.

### 결과
수량 검증 실패 후에도 검증 결과가 정상적으로 전달되어 상품 상세 페이지에서 사용자에게 올바른 피드백을 제공할 수 있게 되었습니다.  
이 경험을 통해 포워드와 리다이렉트는 데이터 전달 방식이 다르므로, 상황에 맞는 객체를 구분해 사용해야 한다는 점을 알게 되었습니다.
