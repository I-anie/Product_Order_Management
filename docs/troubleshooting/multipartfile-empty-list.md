## 상품 수정 시 빈 파일 리스트가 전달되어 이미지 변경 로직이 오동작한 문제

### 문제
상품 수정 기능에서 이미지를 업로드하지 않았는데도 상세 이미지 변경 로직이 실행되는 문제가 있었습니다.  
실제 파일 변경이 없는 상황에서도 이미지 처리 로직이 동작해 의도하지 않은 분기 처리가 발생했습니다.

### 해결
원인을 확인한 결과, `List<MultipartFile>`이 완전히 비어 있지 않고 브라우저 기본값에 의해  
빈 `MultipartFile` 객체가 포함된 상태로 전달되고 있었습니다.  
단순 null 체크나 `isEmpty()`만으로는 실제 업로드 여부를 구분하기 어려워, `stream` 전처리를 통해  
실제 파일 내용이 있는 경우에만 변경 로직을 타도록 수정했습니다.

### 결과
이미지를 업로드하지 않은 경우에는 변경 로직이 실행되지 않고, 실제 파일이 있는 경우에만 이미지 변경이  
정상적으로 동작하도록 개선했습니다.  
이 과정을 통해 파일 업로드 처리에서는 컬렉션 자체의 비어 있음뿐 아니라 각 파일의 실제 내용 유무까지  
함께 확인해야 한다는 점을 알게 되었습니다.
