## Spring Security 설정 순서로 인해 권한 없는 사용자가 화면에 접근 가능했던 문제

### 문제
관리자 전용 화면에 일반 사용자가 접근 가능한 문제가 있었습니다.  
권한 제한을 적용했음에도 일부 URL에서 의도한 접근 제어가 동작하지 않았습니다.

### 해결
원인을 확인한 결과, `SecurityFilterChain`의 `requestMatchers`는 선언 순서대로 적용되는데, 더 넓은 범위의 패턴이 앞에 위치해  
권한 제한이 제대로 적용되지 않고 있었습니다.  
권한이 더 구체적인 URL 패턴을 상단에 배치하도록 순서를 재정리해 접근 제어가 의도한 대로 동작하도록 수정했습니다.

### 결과
관리자 전용 화면에 대한 접근 권한이 정상적으로 제한되었고, 일반 사용자의 비정상 접근을 방지할 수 있게 되었습니다.  
이 경험을 통해 보안 설정에서는 선언 내용뿐 아니라 적용 순서까지 함께 점검해야 한다는 점을 알게 되었습니다.
